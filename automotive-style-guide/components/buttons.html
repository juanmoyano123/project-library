Button Components

Button styles following the automotive design system with precision and bold aesthetics.

Primary Buttons

High-contrast buttons for primary actions. Used for main CTAs and important interactions.

Default
Large
Hover State
<!-- Default Primary Button --> <button class="btn btn-primary">Primary Button</button> <!-- Large Primary Button --> <button class="btn btn-primary btn-large">Large Primary</button> <!-- As Link --> <a href="#" class="btn btn-primary">Primary Link</a>

Secondary Buttons

Outline buttons for secondary actions. Provides visual hierarchy without competing with primary actions.

Default
Large
Hover State
<!-- Default Secondary Button --> <button class="btn btn-secondary">Secondary Button</button> <!-- Large Secondary Button --> <button class="btn btn-secondary btn-large">Large Secondary</button> <!-- As Link --> <a href="#" class="btn btn-secondary">Secondary Link</a>

Button Groups

Multiple buttons arranged together for related actions.

<div style="display: flex; gap: 16px; flex-wrap: wrap;"> <button class="btn btn-primary">Primary Action</button> <button class="btn btn-secondary">Secondary Action</button> </div>

Buttons on Dark Background

Button variations designed for dark backgrounds.

Primary on Dark
Secondary on Dark
<!-- Primary button works the same on dark backgrounds --> <button class="btn btn-primary">Primary Button</button> <!-- Secondary button needs color adjustments --> <button class="btn btn-secondary" style="border-color: #fff; color: #fff;"> Secondary Button </button>

Accessibility Guidelines

  • Minimum size: 44x44px touch target on mobile
  • Color contrast: All buttons meet WCAG AA standards (4.5:1 ratio)
  • Focus states: Visible focus indicator for keyboard navigation
  • Hover states: Clear hover feedback with color transitions
  • Semantic HTML: Use <button> for actions, <a> for navigation
  • ARIA labels: Add descriptive labels for icon-only buttons
